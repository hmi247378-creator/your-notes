# Phase 1（6 周）MVP 开发清单与验收标准

> 本清单按“周/模块”拆分，团队可直接用作迭代看板。目标：完成 **注册登录、标签树、文本录入、关键词分类、小程序基础框架、离线同步**。

## 一、MVP 验收总目标（必须全部满足）
- **分类**：输入一段文字，**< 1 秒**返回 **top3 标签建议 + 置信度**，并支持一键确认/手动调整。
- **标签树**：支持 **1–7 层**标签创建、移动、合并、删除（含必要保护策略）。
- **笔记主流程**：PC 与小程序都能创建/查看/筛选笔记；支持多标签。
- **同步**：PC 与小程序 **< 5 秒**同步可见；小程序离线可记录，联网后自动补偿同步。
- **搜索**：支持全文搜索 + 标签筛选 + 时间范围筛选（MVP 版）。

## 二、周计划拆分（建议）

### Week 0（0.5–1 天）：基建与约定
- **仓库结构**
  - [ ] 建立 `apps/web`、`apps/miniprogram`、`services/api`、`packages/shared`、`docs`
- **本地基础设施**
  - [ ] `docker-compose` 启动 Postgres + Redis（可选 MinIO）
- **协议与规范**
  - [ ] 确认 `docs/API_SPEC.md` 作为联调标准
  - [ ] 确认统一错误码与分页格式
- **验收**
  - [ ] 一条命令可启动 API + Web（小程序可单独启动）

### Week 1：Auth + 数据库建模（最小可用）
- **后端**
  - [ ] 用户表 `users`
  - [ ] `POST /auth/register`、`POST /auth/login`、`GET /auth/me`
  - [ ] 鉴权中间件（保护业务接口）
- **PC Web**
  - [ ] 登录页 + 登录态管理（token 存储策略明确）
- **小程序**
  - [ ] 登录占位页（或临时免登录模式，仅用于本地联调）
- **验收**
  - [ ] 登录后可访问 `GET /tags/tree`（返回空也可）

### Week 2：标签树（核心地基）
- **后端**
  - [ ] `tags` 表（`parentId/path/depth/color`）
  - [ ] `GET /tags/tree`（树形返回）
  - [ ] `POST /tags`（创建，校验 depth ≤ 7）
  - [ ] `PATCH /tags/:id`（重命名/改色/移动，防环 + 深度校验）
  - [ ] `POST /tags/merge`（建议 Phase1 限制叶子合并）
  - [ ] `DELETE /tags/:id`（Phase1 用 block 策略：有子/有笔记则不允许）
- **PC Web**
  - [ ] 左侧标签树展示
  - [ ] 创建子标签、重命名、改色、删除（移动可先“选择父级”）
- **小程序**
  - [ ] 拉取并展示标签（可简化为列表/常用）
- **验收**
  - [ ] 可以创建 7 层标签并正确展示
  - [ ] 移动后子树 path 正确更新

### Week 3：笔记录入（MVP）+ 列表与筛选
- **后端**
  - [ ] `notes`、`note_tags` 表
  - [ ] `POST /notes`（创建）
  - [ ] `GET /notes`（分页 + tagIds + archived + from/to）
  - [ ] `GET /notes/:id`（详情）
  - [ ] `PATCH /notes/:id`（更新内容/标签/归档）
- **PC Web**
  - [ ] 右侧：笔记列表 + 详情查看
  - [ ] 点击标签树节点后筛选列表
  - [ ] 入口输入框（先只创建，不含分类也可）
- **小程序**
  - [ ] 首页输入框创建笔记
  - [ ] 列表页展示最近笔记
- **验收**
  - [ ] PC 创建笔记，小程序能看到；小程序创建笔记，PC 能看到（可先手动刷新）

### Week 4：全文搜索 + 批量操作（最小）
- **后端**
  - [ ] `GET /search?q=...`（Postgres FTS）
  - [ ] `POST /notes/batch`（批量改标签/归档/删除，至少支持前两项）
- **PC Web**
  - [ ] 搜索框（全文）
  - [ ] 列表多选 + 批量改标签/归档
- **小程序**
  - [ ]（可选）搜索入口（Phase1 不做也可，优先离线同步）
- **验收**
  - [ ] 能搜索到包含关键词的笔记
  - [ ] 批量改标签后筛选结果正确

### Week 5：智能分类（Phase1）+ 反馈闭环
- **后端**
  - [ ] `POST /classify`（返回 top3 + level）
  - [ ] `classification_suggestions`、`classification_feedback` 表
  - [ ] `POST /feedback`（保存用户修正）
  - [ ] 标签画像更新任务（可选异步 job）
- **PC Web**
  - [ ] 输入后展示建议标签 top3（高/中/低）
  - [ ] 一键确认 / 手动调整（树选择器）
  - [ ] 调整后上报 feedback
- **小程序**
  - [ ] 提交文本后展示建议 top3（简化版 UI）
  - [ ] 一键确认 / 手动调整（可先做“选择一个标签”）
- **验收**
  - [ ] `POST /classify` 在常规数据量下 < 1 秒
  - [ ] 反馈数据落库可查询（为 Phase2 做准备）

### Week 6：离线同步（小程序重点）+ 稳定性 + 上线准备
- **后端**
  - [ ] `change_log` 表
  - [ ] `POST /sync/push`（幂等、批量）
  - [ ] `GET /sync/pull?since=...`（增量）
  - [ ] 关键指标埋点（分类命中率/修正率/同步成功率）
  - [ ] 每日备份脚本（MVP：数据库备份文件）
- **小程序**
  - [ ] `pendingChanges[]` 本地队列（无网可创建/更新笔记）
  - [ ] 联网自动 push + pull
  - [ ] 同步状态提示（离线/同步中/失败重试）
- **PC Web**
  - [ ] 定时 pull（或刷新触发）展示跨端变更
- **验收**
  - [ ] 无网创建 3 条笔记，联网后自动同步到服务端并在 PC 可见
  - [ ] 重复 push 不产生重复笔记（幂等有效）

## 三、关键测试用例（建议在 Phase1 结束前全部跑通）

### 3.1 标签树
- [ ] 创建：1–7 层正常；第 8 层提示并阻止
- [ ] 移动：移动到子树中被阻止；移动后子树 path 正确
- [ ] 合并：合并后笔记关联迁移正确（无重复）
- [ ] 删除：有子标签或有关联笔记时阻止删除（Phase1 策略）

### 3.2 笔记
- [ ] PC 创建笔记，5 秒内小程序可见（或下一次 pull 可见）
- [ ] 小程序创建笔记，PC 可见
- [ ] 多标签绑定与筛选正确
- [ ] 批量改标签/归档后列表与筛选正确

### 3.3 分类
- [ ] top3 建议返回稳定；置信度 level 与分差映射符合预期
- [ ] 用户一键确认：记录“接受 top1/top2/top3”
- [ ] 用户手动调整：反馈落库（before/after）

### 3.4 同步与离线
- [ ] 离线写入：本地队列不丢失
- [ ] 断网/弱网：自动重试，不阻塞用户继续记录
- [ ] 幂等：同一 `clientMutationId` 重放不重复生效

## 四、上线前最小风控清单

- [ ] 基本审计日志（用户、时间、操作类型）
- [ ] 数据备份可恢复演练（至少一次）
